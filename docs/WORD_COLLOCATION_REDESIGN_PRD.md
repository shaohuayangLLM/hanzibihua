# 词语搭配功能重规划 PRD（全量重构版）

- 项目: K12-Education
- 模块: 词语搭配（`/word-collocation`）
- 版本: v2.0.0-draft
- 日期: 2026-02-12
- 状态: Draft（模块已下线）

## 1. 决策与现状

1. 词语搭配已全量下线，入口禁用，页面改为“重构中”。
2. 原题库存在系统性歧义风险：
- 题干语境过短。
- 干扰项语义可成立。
- 仅靠字符串命中不能保证教学唯一正确。

## 2. 重构目标

1. 每题在小学低年级语义下必须“唯一正确”。
2. 题目可解释：错因明确、可复述规则。
3. 上线题全部可追溯：来源、审核、门禁结果完整记录。

## 3. 新题型设计（V3）

## 3.1 核心题型

1. 语境填空题（唯一答案）
- 题干必须包含约束锚点（时间/地点/动作对象/结果）。
- 禁止使用宽泛模板句：如“我喜欢___”。

2. 搭配辨析题（对比最小差异）
- 4 选 1，但仅允许 1 个选项语义+语法同时成立。
- 其余 3 项必须覆盖不同错误类型：词性错误、语义冲突、搭配习惯错误。

3. 场景选择题（图/文情境）
- 给定场景描述，选最合适搭配。
- 每个干扰项都需附带“为何不成立”的结构化理由。

## 3.2 出题约束（硬规则）

1. 选项同槽位：词性、句法位置一致。
2. 选项去重后必须为 4。
3. 每题至少一个“后锚点”或“前后双锚点”。
4. 干扰项禁止同义近义并列（如“非常美丽/非常漂亮”）。
5. 结果补语、趋向补语、地点短语题默认高风险，需人工审核通过才可发布。

## 4. 数据模型

```ts
interface CollocationQuestionV3 {

    source: "manual" | "curated";

  uniquenessCheck: {

  };
}
```

## 5. 质量门禁

上线必须同时满足：

1. 自动校验通过
- 结构完整性。
- 词法唯一命中。
- 语境信号强度达标。

2. 教研审核通过
- “唯一正确”人工复核。
- 解释可理解性复核。

3. 回归测试通过
- 页面渲染。
- 答题流程。
- 筛选/计分/重置。

## 6. 研发实施计划

## 阶段 A：题库冻结与基线（0.5 天）

1. 冻结旧题库，不再新增旧格式题。
2. 产出 `draft` 题清单与原因聚类。

## 阶段 B：V3 题库骨架（1 天）

1. 新建 `wordCollocationV3.ts` 与类型。
2. 完成首批 30 题人工重编（覆盖 7 类）。

## 阶段 C：审核与门禁（1 天）

1. 接入 `semanticReviewPassed` 校验位。
2. 增加发布脚本：仅导出 `published` 题。

## 阶段 D：灰度回放（0.5 天）

1. 仅开放 30 题灰度。
2. 监控歧义反馈率与复错率。

## 阶段 E：全量恢复（按周迭代）

1. 每周增加 20~30 题。
2. 每批先审核后发布。

## 7. 验收标准

1. 上线题“唯一正确”人工抽检通过率 >= 99%。
2. 歧义反馈率 < 0.5%。
3. 首答错误后二次错误率下降 >= 20%。

## 8. 风险与对策

1. 题量下降
- 对策：先保证质量，分批恢复，不追求一次性回满。

2. 人工审核成本高
- 对策：模板化错误类型与解释文案，减少重复劳动。

3. 线上再次出现歧义
- 对策：即时降级到 `draft`，24 小时内替换题目。

## 9. 当前执行结论

1. 功能已全量下线，避免继续误导。
2. 进入 V3 重构阶段，下一步从“首批 30 题人工重编”启动。
