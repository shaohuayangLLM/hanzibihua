## 问题诊断

### 核心问题
**拼音组合页面朗读拼音音标（如 "bā", "bá"）时，TTS 引擎无法准确识别声调符号**

### 原因分析

1. **TTS 引擎设计限制**
   - 阿里云 TTS（Qwen-TTS）和 Web Speech API 是为**朗读汉字**设计的
   - 它们**不是为朗读拼音音标**设计的
   - 声调符号（ā á ǎ à）可能被识别为特殊字符，无法正确发音

2. **错误实现**
   ```typescript
   // ❌ 错误：朗读拼音音标
   speak("bā");  // TTS 可能读成 "b a" 或无法识别
   speak("bā bá bǎ bà");  // 更加混乱
   ```

3. **正确实现**
   ```typescript
   // ✅ 正确：朗读对应的汉字
   speak("八");  // TTS 正确读出汉字发音
   speak("八 拔 把 爸");  // 清晰朗读四个声调
   ```

## 受影响组件

### 1. CombinationCardNew.tsx（新版卡片组件）✅ 已修复

**修复内容**：

#### 修复前（错误）
```typescript
// ❌ 播放所有声调 - 朗读拼音音标
const handlePlayAll = (e: React.MouseEvent) => {
  e.stopPropagation();
  const allTones = item.tones.map(t => t.combination).join(" ");
  speak(allTones);  // "bā bá bǎ bà" 无法准确发音
};

// ❌ 播放单个声调 - 朗读拼音音标
const handlePlayTone = (combination: string, e: React.MouseEvent) => {
  e.stopPropagation();
  speak(combination);  // "bā" 无法准确发音
};
```

#### 修复后（正确）
```typescript
// ✅ 播放所有声调 - 朗读汉字
const handlePlayAll = (e: React.MouseEvent) => {
  e.stopPropagation();
  // 收集所有声调的例字
  const allChars = item.tones
    .filter(t => t.examples[0])
    .map(t => t.examples[0].char)
    .join(" ");
  speak(allChars);  // "八 拔 把 爸" 准确发音
};

// ✅ 播放单个声调 - 朗读例词
const handlePlayTone = (tone: typeof item.tones[0], e: React.MouseEvent) => {
  e.stopPropagation();
  if (tone.examples[0]) {
    // 朗读例词（如"八个"），而不是拼音（如"bā"）
    speak(tone.examples[0].word || tone.examples[0].char);
  }
};
```

**效果对比**：
| 操作 | 修复前 | 修复后 |
|------|--------|--------|
| 点击声调行 | 朗读 "bā"（不准确） | 朗读 "八个"（准确） |
| 点击"播放所有声调" | 朗读 "bā bá bǎ bà"（混乱） | 朗读 "八 拔 把 爸"（清晰） |

## 技术原理

### TTS 引擎工作方式

#### 阿里云 Qwen-TTS
- **训练数据**：大量汉字语音数据
- **输入格式**：中文汉字文本
- **输出**：对应的语音音频
- **声调识别**：基于汉字的固有声调，而非拼音符号

#### Web Speech API
- **语音库**：操作系统提供的 TTS 语音库
- **中文支持**：`zh-CN` 语言设置
- **工作原理**：识别汉字并查询内置发音数据库
- **拼音支持**：**不支持拼音音标发音**

### 为什么拼音音标无法准确朗读

1. **字符识别问题**
   - `ā` 不是标准汉字，是 Unicode 拉丁字母扩展字符
   - TTS 引擎可能将其识别为 "a + 声调符号"，而非完整音节

2. **声调符号处理**
   - 声调符号（ˉ ˊ ˇ ˋ）是变音符号，不是独立字符
   - TTS 可能忽略或错误处理这些符号

3. **训练数据缺失**
   - TTS 引擎的训练数据主要是汉字朗读
   - 拼音朗读数据极少或没有

## 最佳实践

### 拼音朗读场景分类

| 场景 | 内容 | 朗读对象 | 示例 |
|------|------|---------|------|
| **声母韵母学习** | 单个拼音音素 | 拼音本身 | "b" → 朗读 "b" |
| **整体认读音节** | 完整但无声调 | 拼音本身 | "zhi" → 朗读 "zhi" |
| **带声调拼音** | 完整音节+声调 | **对应汉字** | "bā" → 朗读 "八" |
| **拼音组合** | 多个带声调拼音 | **对应汉字** | "bā bá" → 朗读 "八 拔" |

### 代码实现模式

```typescript
// 模式1：朗读基础拼音（声母、韵母）
speak(pinyin);  // "b", "a", "zh"

// 模式2：朗读带声调拼音组合
speak(char);  // "八"（而非 "bā"）

// 模式3：朗读例词
speak(word || char);  // "八个"（优先）或 "八"（降级）

// 模式4：连续朗读多个声调
speak(chars.join(" "));  // "八 拔 把 爸"
```
